Detailed Description

SafeSocket TCP client with real-time messaging and file transfer.

A Client instance represents one connection to a SafeSocket server. It spawns a background receive thread (recv_loop) that runs concurrently with the main CLI thread (run_cli). All shared state between the two threads is protected by m_peers_mutex.

Thread Safety

m_peers and m_my_id are written only by the receive thread. m_connected and m_running are std::atomic<bool> and safe to read from any thread without a lock.

bool Client::connect ( const std::string & host,
int port )

Establish a TCP connection to a SafeSocket server.

Establish a TCP connection to a SafeSocket server and begin receiving.

Creates the download directory, resolves the nickname from g_config.nickname, calls net_connect(), starts the receive thread, and sends the MSG_CONNECT handshake (including the optional access key).

Parameters
hostDotted-decimal IPv4 address of the server.
portTCP port the server is listening on.
Returns
true if the connection was established and the handshake sent, false on failure.

Steps:

  1. Create the download directory via mkdir_compat().
  2. Copy the nickname from g_config.nickname.
  3. Call net_connect() to obtain a connected socket.
  4. Launch recv_loop() on m_recv_thread.
  5. Build and send the MSG_CONNECT handshake packet containing the nickname and (when non-empty) the access key.
  6. Print the connection banner.
Parameters
hostDotted-decimal IPv4 address of the server.
portTCP port the server is listening on.
Returns
true on success, false if net_connect() failed.

◆ run_cli()

void Client::run_cli ( )

Run the interactive command-line interface until the user quits.

Block the calling thread running the user command-line interface.

Blocks the calling thread reading commands from stdin. Supported commands include /broadcast, /msg, /msgn, /list, /sendfile, /sendfileserver, /nick, /myid, /config, /set, and /quit.

Plain text entered without a leading / is sent as a broadcast message. Returns when the user disconnects or stdin reaches EOF.

Reads commands from stdin until the user types /quit or /exit, or until the connection is lost or stdin reaches EOF. Each iteration:

  1. Prints the "[nickname]> " prompt.
  2. Reads a line with std::getline().
  3. If the line does not start with '/', sends it as a broadcast message.
  4. Otherwise, parses the leading command token (lowercased) and dispatches to the appropriate Client method or Config operation.

Supported commands: /broadcast, /msg, /msgn, /list, /sendfile, /sendfileserver, /nick, /myid, /config, /confighelp, /set, /setconfig, /saveconfig, /loadconfig, /quit (/exit, /disconnect), /help (/?).
