Detailed Description

Aggregates all runtime-configurable parameters for SafeSocket.

Both the Server and Client read from the single global instance g_config. Fields are grouped into logical sections that mirror the sections written by Config::save().

Thread-safety note: concurrent reads are safe, but writes via Config::set() are not protected by a mutex. Runtime changes should only be made from the CLI thread.

◆ load()

bool Config::load ( const std::string & path)

Load configuration from a plain-text key=value file.

Parse a configuration file and apply the contained key-value pairs.

Lines beginning with '#' or ';' are treated as comments and skipped. Each line is expected to have the form key=value; everything after an inline " #" comment marker is stripped. Unknown keys are silently ignored (the underlying set() call will emit a warning to stderr).

Parameters
pathFilesystem path of the configuration file to read.
Returns
true if the file was opened and parsed successfully, false if the file could not be opened.

The parser handles:

  • Comment lines (starting with '#' or ';') — skipped.
  • Blank lines — skipped.
  • key=value lines — the key and value are trimmed, then passed to Config::set().
  • Inline comments after " #" in the value portion — stripped.
  • Section headers (e.g. [network]) — silently ignored.

Unknown keys produce a warning via Config::set() but do not abort the load.

Parameters
pathFilesystem path of the configuration file to read.
Returns
true if the file was opened and processed; false if the file could not be opened.

void Config::set ( const std::string & key,
const std::string & value )

Set a single configuration field by name, parsing the value string.

Set a configuration field by name.

Key matching is case-insensitive. Boolean values accept "true", "1", "yes", or "on" for true; anything else is false. Integer values are parsed with std::stoi(); a parse failure falls back to the field's current value (or a hardcoded default).

Parameters
keyCase-insensitive configuration field name (e.g. "port").
valueString representation of the new value to assign.
Note
Emits a warning to stderr for unrecognised keys.

The key lookup is case-insensitive. Each recognised key maps directly to a member field; the string value is converted to the appropriate type using parse_int(), parse_bool(), or encrypt_type_from_string() as needed.

If key does not match any known field a warning is emitted to stderr and no change is made.

Parameters
keyCase-insensitive name of the configuration field to update.
valueString representation of the new value.
