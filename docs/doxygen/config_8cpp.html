Detailed Description

Implementation of configuration load, save, set, get, and print for SafeSocket.

Defines the global g_config instance and provides all methods declared in config.hpp. Configuration can be persisted to and loaded from a plain-text INI-style file, and individual keys can be mutated at runtime via Config::set().

File Format

The configuration file uses a simple key = value syntax:

# This is a comment
port = 9000
encrypt_type = RC4
encrypt_key = mysecret # inline comment
EncryptType::RC4
@ RC4
RC4 stream cipher; key-scheduled pseudo-random XOR, stronger than XOR/Vigenčre.
Definition crypto.hpp:53

Lines beginning with '#' or ';' are comments. Inline comments after a space-hash (" #") sequence are stripped from values. Section headers ([network] etc.) are written by Config::save() for human readability but are ignored by the parser — only key=value lines matter.
