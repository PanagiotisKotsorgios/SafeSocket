Detailed Description

Shared packet structures, message types, and protocol constants for SafeSocket.

This header defines the complete wire protocol used by SafeSocket for all client-server communication. It includes message type enumerations, binary packet structures, special IDs, protocol constants, and inline utility functions for building packets.

All multi-byte fields are stored in host byte order unless explicitly noted. Consumers are responsible for byte-order conversions where interoperability across different architectures is required.

#define BROADCAST_ID   0xFFFFFFFFu

Sentinel target ID indicating a broadcast to all connected clients.

When target_id is set to this value the server should relay the packet to every client currently in the session.

◆ SERVER_ID

Used in the sender_id or target_id fields of PacketHeader to indicate that the server is the originator or the sole recipient.

◆ MsgType

enum MsgType : uint32_t

Enumeration of all message types used in the SafeSocket protocol.

Each value identifies the purpose of a packet and determines how the receiver should interpret the payload. The underlying type is uint32_t to match the msg_type field of PacketHeader.

Builds a complete raw packet (header + payload) in a byte vector.

The returned vector contains a fully formed PacketHeader followed immediately by a copy of data. No encryption is applied here; callers that need encryption should use net_send_packet() instead.

Parameters
typeMsgType value identifying the packet purpose.
senderNumeric ID of the originating client or SERVER_ID.
targetNumeric ID of the intended recipient or BROADCAST_ID.
dataPointer to the payload bytes; may be nullptr when data_len is 0.
data_lenNumber of payload bytes pointed to by data.
Returns
Byte vector containing the complete packet ready for transmission.

Convenience wrapper around make_packet() for text payloads.

Converts text to a raw byte range and delegates to make_packet(). The string is not null-terminated in the packet; the receiver reconstructs it from data_len in the header.
