Detailed Description

Implementation of the cross-platform socket abstraction layer.

Provides the concrete implementations for all functions declared in network.hpp. Platform-specific code paths are wrapped in #ifdef _WIN32 / #else blocks so that a single source file compiles cleanly on both Windows (Winsock2) and POSIX (BSD sockets) targets.

Key implementation notes:

  • net_send_raw() and net_recv_raw() loop internally to handle partial sends/receives, guaranteeing that exactly the requested number of bytes is transferred or an error is returned.
  • net_connect() uses a non-blocking connect() + select() pattern to honour the caller-supplied timeout without blocking indefinitely.
  • net_send_packet() encrypts the payload before transmission.
  • net_recv_packet() decrypts the payload after reception and validates the magic number in the header.

sock_t net_connect ( const std::string & host,
int port,
int timeout_sec )

Create a TCP socket and connect it to a remote server with a timeout.

Create a TCP socket and connect it to a remote host.

Algorithm:

  1. Create an AF_INET socket and set TCP_NODELAY.
  2. Switch the socket to non-blocking mode.
  3. Call connect(); on POSIX/Windows this returns immediately with EINPROGRESS / WSAEWOULDBLOCK for a connection in progress.
  4. Use select() to wait up to timeout_sec seconds for writability, which signals that the connection completed or failed.
  5. Use getsockopt(SO_ERROR) to confirm the connection succeeded.
  6. Restore blocking mode before returning.
Parameters
hostDotted-decimal IPv4 address of the server.
portTCP port number of the server.
timeout_secMaximum seconds to wait for the connection to be established.
Returns
A valid connected socket, or INVALID_SOCK on failure or timeout.

Initialise the networking subsystem.

Create a TCP listening socket bound to the specified address and port.

Create a TCP listening socket bound to the given address and port.

Steps:

  1. Create an AF_INET / SOCK_STREAM / IPPROTO_TCP socket.
  2. Apply SO_REUSEADDR and TCP_NODELAY.
  3. Populate a sockaddr_in and call bind().
  4. Call listen() with the given backlog.

An empty host or "0.0.0.0" binds to all interfaces (INADDR_ANY).

Parameters
hostIPv4 address string to bind to.
portTCP port number.
backlogMaximum pending-connection queue length.
Returns
A valid listening socket, or INVALID_SOCK on any failure.

bool net_recv_packet ( sock_t s,
PacketHeader & hdr,
std::vector< uint8_t > & payload,
EncryptType enc,
const std::string & key )

Receive and deserialize a complete SafeSocket packet.

Steps:

  1. Receive exactly HEADER_SIZE bytes into hdr via net_recv_raw().
  2. Validate hdr.magic against MAGIC; on mismatch print an error and return false — this typically indicates a desynchronised stream.
  3. Resize payload to hdr.data_len and receive that many bytes.
  4. Decrypt the payload in-place with crypto_decrypt().
Parameters
[in]sConnected socket to read from.
[out]hdrPopulated with the received packet header.
[out]payloadPopulated with the decrypted payload bytes.
[in]encDecryption algorithm (must match the sender's encryption).
[in]keyDecryption passphrase (must match the sender's key).
Returns
true on success, false on I/O error, connection closure, or magic-number mismatch.

Serialize and send a complete SafeSocket packet.

If data is non-null and data_len is greater than zero, the payload is copied into a local std::vector and encrypted in-place with crypto_encrypt(). The PacketHeader is then populated (including the post-encryption data_len) and transmitted via net_send_raw(), followed by the payload.
