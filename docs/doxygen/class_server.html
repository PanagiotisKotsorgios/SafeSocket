Detailed Description

TCP chat-and-file-transfer server managing multiple concurrent clients.

The server spawns one thread per connected client to handle receive loops, one accept thread to listen for incoming connections, and an optional keepalive thread to send periodic pings. All access to the client map is protected by m_clients_mutex.

Thread Safety

Public methods that touch the client map acquire m_clients_mutex internally. Do not call public methods from within a context that already holds this lock.

◆ run_cli()

void Server::run_cli ( )

Run the interactive command-line interface until the operator quits.

Block the calling thread running the operator command-line interface.

Blocks the calling thread reading from stdin. Supports commands such as /list, /broadcast, /msg, /kick, /sendfile, /stats, /config, and /quit. Returns when /quit (or equivalent) is entered or stdin reaches EOF.

Reads commands from stdin in a loop until the operator types /quit, /stop, or /exit, or until stdin reaches EOF. Each iteration:

  1. Prints the "[SERVER]> " prompt.
  2. Reads a line with std::getline().
  3. Trims whitespace and skips blank input.
  4. Parses the leading command token (normalised to lowercase).
  5. Dispatches to the appropriate Server method or Config operation.

Supported commands: /list, /msg, /msgn, /broadcast, /kick, /kickn, /sendfile, /sendfileall, /stats, /config, /confighelp, /set, /setconfig, /saveconfig, /loadconfig, /quit (/stop, /exit), /help (/?).

bool Server::start ( const std::string & host,
int port )

Bind to the given address and start accepting client connections.

Initialise and start the server.

Creates the download directory, opens the log file if configured, creates the listening socket via net_listen(), and launches the accept and (optionally) keepalive threads.

Parameters
hostIPv4 address to bind to; "0.0.0.0" binds to all interfaces.
portTCP port to listen on.
Returns
true on successful bind and listen, false on failure.

Performs the following steps in order:

  1. Create the download directory with mkdir_compat().
  2. Open the log file if g_config.log_to_file is true.
  3. Call net_listen() to bind and begin listening.
  4. Set m_running to true.
  5. Launch accept_loop() on m_accept_thread.
  6. Conditionally launch keepalive_loop() on m_keepalive_thread.
  7. Print the startup banner.
Parameters
hostIPv4 address to bind to.
portTCP port to listen on.
Returns
true on success, false if net_listen() failed.

void Server::stop ( )

Gracefully shut down the server.

Gracefully stop the server and release all resources.

Closes the listening socket, signals all client threads to stop, joins the accept and keepalive threads, and releases all client resources. Safe to call multiple times; subsequent calls are no-ops.

Sets m_running to false, closes the listening socket to unblock the accept thread, marks every client as not alive, closes all client sockets, and joins both the accept and keepalive threads. Finally iterates over all client entries to join their threads and free heap memory.

Safe to call multiple times; the second and subsequent calls are no-ops.
