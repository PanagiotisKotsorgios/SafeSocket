Detailed Description

Implementation of SafeSocket's built-in symmetric encryption algorithms.

Provides concrete implementations for the four algorithms declared in crypto.hpp. All algorithms are implemented from scratch with no dependency on OpenSSL or any other external cryptographic library.

Algorithm Notes

XOR (xor_crypt) Simple repeating-key XOR. The key byte at position iklen is XOR-ed with the plaintext byte at position i. The operation is its own inverse, so the same function is used for both encryption and decryption. Security: trivially broken by known-plaintext; suitable only for basic obfuscation.

Vigenčre (vigenere_encrypt / vigenere_decrypt) Byte-level Vigenčre operating on the full 0–255 range (unlike the classic text-only variant). Encryption adds the key byte modulo 256; decryption subtracts it modulo 256. Stronger than XOR against simple frequency analysis, but still vulnerable to the Kasiski examination.

RC4 (RC4State, rc4_crypt) Standard RC4 (also known as ARC4) stream cipher. The key-scheduling algorithm (KSA) initialises the 256-byte permutation array, and the pseudo-random generation algorithm (PRGA) produces a keystream that is XOR-ed with the data. RC4 is symmetric: the same PRGA sequence is generated from the same key, so the same function encrypts and decrypts. Security: RC4 has known weaknesses (biased output bytes, susceptibility to related-key attacks) and should not be used in new security-critical systems, but is significantly stronger than XOR or Vigenčre for casual use.

Decrypt a byte buffer in-place using the specified algorithm.

Decrypt a byte buffer in-place using the specified algorithm and key.

Reverses the transformation applied by crypto_encrypt() when called with the same type and key.

For symmetric algorithms (XOR, RC4) the same internal function is called as during encryption — the XOR with an identical keystream restores the original data. For Vigenčre the dedicated subtraction-based vigenere_decrypt() is used. For EncryptType::NONE the buffer is left completely unchanged.

Parameters
[in,out]bufBuffer to decrypt; modified in-place.
[in]typeAlgorithm to reverse; must match the value used for encryption.
[in]keyKey/passphrase; must match the value used for encryption.

< XOR is self-inverse.

< RC4 is self-inverse.
