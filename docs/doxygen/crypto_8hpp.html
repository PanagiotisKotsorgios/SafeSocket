Detailed Description

Lightweight symmetric encryption/decryption support for SafeSocket.

Provides four selectable algorithms that require no external libraries:

  • NONE — plaintext; no transformation applied.
  • XOR — repeating-key XOR; fast but weak, suitable for obfuscation only.
  • VIGENERE — byte-level Vigenère cipher operating across the full 0–255 range.
  • RC4 — RC4 stream cipher; stronger key-based encryption for practical use.

All algorithms operate in-place on a std::vector<uint8_t> buffer, so no additional heap allocation is required beyond the buffer itself.

Symmetry

XOR and RC4 are self-inverse: calling crypto_encrypt() and then crypto_decrypt() (or vice versa) with the same key and algorithm restores the original data. Vigenère uses separate encrypt/decrypt paths (addition vs. subtraction modulo 256).

Usage Example

std::vector<uint8_t> data = {'H','e','l','l','o'};
crypto_encrypt(data, EncryptType::RC4, "mypassword");
// ... transmit data ...
crypto_decrypt(data, EncryptType::RC4, "mypassword");
// data is restored to {'H','e','l','l','o'}

enum class EncryptType
strong

Selects the symmetric encryption algorithm to apply to packet payloads.

The underlying type is int to allow it to be stored in configuration files and transmitted as a simple integer where needed.

Enumerator
NONE 

No encryption; data is transmitted as plaintext.

XOR 

Repeating-key XOR; very fast, suitable for lightweight obfuscation only.

VIGENERE 

Byte-level Vigenère cipher operating across the full 0–255 range.

RC4 

RC4 stream cipher; key-scheduled pseudo-random XOR, stronger than XOR/Vigenère.

Decrypt a byte buffer in-place using the specified algorithm and key.

Reverses the transformation applied by crypto_encrypt() when called with the same type and key. For symmetric algorithms (XOR, RC4) this function is identical to crypto_encrypt(); for Vigenère a distinct subtraction-based path is used.

Note
For EncryptType::NONE this function returns immediately without touching buf.

Decrypt a byte buffer in-place using the specified algorithm and key.

Reverses the transformation applied by crypto_encrypt() when called with the same type and key.

For symmetric algorithms (XOR, RC4) the same internal function is called as during encryption — the XOR with an identical keystream restores the original data. For Vigenère the dedicated subtraction-based vigenere_decrypt() is used. For EncryptType::NONE the buffer is left completely unchanged.

◆ encrypt_type_from_string()

EncryptType encrypt_type_from_string ( const std::string & s)

Parse a string into an EncryptType value.

Comparison is case-insensitive; "rc4", "RC4", and "Rc4" all map to EncryptType::RC4. Any unrecognised string returns EncryptType::NONE rather than throwing an exception, so configuration files with typos fall back to plaintext silently.

Parameters
sThe string to parse (e.g. "RC4", "xor", "VIGENERE").
Returns
The matching EncryptType, or EncryptType::NONE if unrecognised.
