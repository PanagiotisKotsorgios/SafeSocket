Detailed Description

Cross-platform socket abstraction layer for SafeSocket.

Provides a unified API for TCP networking on both Windows (Winsock2) and POSIX (BSD sockets) platforms. All platform-specific preprocessor guards, type aliases, and macro definitions are contained in this header so that the rest of the codebase can remain platform-agnostic.

Key responsibilities:

  • Platform detection and conditional inclusion of socket headers.
  • Type aliases (sock_t, INVALID_SOCK, SOCK_ERR, sock_close).
  • Networking lifecycle functions: net_init() and net_cleanup().
  • Server-side helpers: net_listen() and net_accept().
  • Client-side helper: net_connect().
  • Socket option setters: net_set_nonblocking(), net_set_nodelay(), net_set_reuseaddr().
  • Reliable byte-stream I/O: net_send_raw() and net_recv_raw().
  • Protocol-aware framed I/O: net_send_packet() and net_recv_packet().
  • Diagnostics: net_error_str() and fmt_bytes().

Create a TCP socket and connect it to a remote host.

Uses a non-blocking connect() combined with select() to enforce the specified connection timeout. The socket is restored to blocking mode before being returned.

Parameters
hostDotted-decimal IPv4 address of the server to connect to.
portTCP port number of the server.
timeout_secMaximum seconds to wait for the connection to complete (default 10).
Returns
A valid connected socket on success, or INVALID_SOCK on failure.

Create a TCP socket and connect it to a remote host.

Algorithm:

  1. Create an AF_INET socket and set TCP_NODELAY.
  2. Switch the socket to non-blocking mode.
  3. Call connect(); on POSIX/Windows this returns immediately with EINPROGRESS / WSAEWOULDBLOCK for a connection in progress.
  4. Use select() to wait up to timeout_sec seconds for writability, which signals that the connection completed or failed.
  5. Use getsockopt(SO_ERROR) to confirm the connection succeeded.
  6. Restore blocking mode before returning.
Parameters
hostDotted-decimal IPv4 address of the server.
portTCP port number of the server.
timeout_secMaximum seconds to wait for the connection to be established.
Returns
A valid connected socket, or INVALID_SOCK on failure or timeout.

Initialise the networking subsystem.

On Windows this calls WSAStartup() to load Winsock 2.2. On POSIX it installs SIG_IGN for SIGPIPE so that writes to closed sockets produce an error return rather than terminating the process.

Must be called once before any other net_* function.

Returns
true on success, false if initialisation failed (Windows only).

Initialise the networking subsystem.

On Windows: calls WSAStartup(MAKEWORD(2,2), &wsa) to load Winsock 2.2. Returns false and prints an error if WSAStartup fails.

On POSIX: installs SIG_IGN for SIGPIPE so that writes to a closed peer socket return EPIPE rather than delivering an unhandled signal that would terminate the process.

Returns
true on success; false only on Windows when WSAStartup fails.

Create a TCP listening socket bound to the given address and port.

Steps:

  1. Create an AF_INET / SOCK_STREAM / IPPROTO_TCP socket.
  2. Apply SO_REUSEADDR and TCP_NODELAY.
  3. Populate a sockaddr_in and call bind().
  4. Call listen() with the given backlog.

An empty host or "0.0.0.0" binds to all interfaces (INADDR_ANY).

Receive and deserialize a complete SafeSocket packet.

Reads a PacketHeader, validates the magic number, reads the payload indicated by data_len, and then decrypts it using crypto_decrypt().

Parameters
[in]sConnected socket to read from.
[out]hdrPopulated with the received PacketHeader.
[out]payloadPopulated with the decrypted payload bytes.
[in]encDecryption algorithm to apply (must match the sender's encryption).
[in]keyPassphrase used for decryption (must match the sender's key).
Returns
true on success, false on I/O error, disconnection, or bad magic.

Steps:

  1. Receive exactly HEADER_SIZE bytes into hdr via net_recv_raw().
  2. Validate hdr.magic against MAGIC; on mismatch print an error and return false — this typically indicates a desynchronised stream.
  3. Resize payload to hdr.data_len and receive that many bytes.
  4. Decrypt the payload in-place with crypto_decrypt().
Parameters
[in]sConnected socket to read from.
[out]hdrPopulated with the received packet header.
[out]payloadPopulated with the decrypted payload bytes.
[in]encDecryption algorithm (must match the sender's encryption).
[in]keyDecryption passphrase (must match the sender's key).
Returns
true on success, false on I/O error, connection closure, or magic-number mismatch.
