Detailed Description

Holds all state associated with a single connected client.

Instances are heap-allocated by the accept loop and stored in the server's client map. Each instance owns the client's socket and receive thread for its lifetime.

Note
The thread member is detach()ed after creation to avoid blocking the accept loop. Clean-up on disconnect relies on the alive flag.
