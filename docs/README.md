
# Project Documentation

This directory contains the full documentation for the project, including
user guides, developer documentation, architecture information, and
generated API documentation.

The documentation is organized into several sections to make it easier
to navigate and maintain.

---

# Documentation Overview

## App Documentation

Detailed app documentation generated with **Doxygen** is available in:

```
docs/doxygen/
```

If the HTML documentation is generated, open:

```
docs/doxygen/html/index.html
```

in a web browser.

This documentation includes:

* Class documentation
* Function reference
* File documentation
* Call graphs and dependency graphs

---

## Build Instructions

Information on how to build the project on different platforms.

```
docs/build/
```

---

## Usage Guide

Instructions and examples on how to use the application.

```
docs/usage/
```

---

## Architecture

High-level description of the application architecture, components,
and design decisions.

```
docs/architecture/
```

---

# Notes

* Source code documentation comments are written using the **Doxygen format**
* The generated documentation should not be manually modified
* Changes should be made in the source code comments instead
